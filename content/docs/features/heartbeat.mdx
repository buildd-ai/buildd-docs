# Heartbeat Monitoring

Heartbeat monitoring lets you define a checklist of items that agents verify on a recurring schedule. Each heartbeat creates a planning task that evaluates the checklist and reports findings.

## Setup

### Configure the Checklist

Navigate to **Settings** in the Buildd dashboard. Each workspace has a **Heartbeat Checklist** section where you can:

- Add monitoring items (e.g., "API response times under 200ms", "No error spikes in logs")
- Remove items with the delete button
- Items persist immediately on save

### Create a Heartbeat Schedule

Use the MCP server to create a recurring heartbeat:

```
buildd action=create_heartbeat params={
  cronExpression: "*/30 * * * *",
  name: "Production Health Check"
}
```

This creates a [schedule](/docs/features/schedules) that spawns a planning-mode task every 30 minutes. The task title defaults to **"Heartbeat: check monitored items"** with priority 3.

The default cron is `*/30 * * * *` (every 30 minutes) if not specified.

## How It Works

1. The schedule triggers and creates a planning-mode task
2. A worker claims the task and reviews the heartbeat checklist
3. The worker investigates each item and submits a plan/report
4. You review findings in the dashboard or approve/reject via the plan review panel

## Managing via MCP

| Action | Params | Description |
|--------|--------|-------------|
| `manage_heartbeat` | `action: "get"` | Retrieve current checklist |
| `manage_heartbeat` | `action: "set", checklist: [...]` | Replace the checklist |
| `create_heartbeat` | `cronExpression?, name?` | Create a heartbeat schedule |

### Examples

```
# View current checklist
buildd action=manage_heartbeat params={ action: "get" }

# Update checklist
buildd action=manage_heartbeat params={
  action: "set",
  checklist: [
    "API endpoints responding under 500ms",
    "Error rate below 1%",
    "Database connections stable",
    "SSL certificates not expiring within 30 days"
  ]
}

# Create hourly heartbeat
buildd action=create_heartbeat params={
  cronExpression: "0 * * * *",
  name: "Hourly Infrastructure Check"
}
```

## API Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/workspaces/{id}/heartbeat` | Get the checklist |
| `PATCH` | `/api/workspaces/{id}/heartbeat` | Update the checklist |

### Request/Response

```bash
# Get checklist
curl https://buildd.dev/api/workspaces/{id}/heartbeat \
  -H "Authorization: Bearer bld_xxx"

# Response
{ "checklist": ["API health", "Error rate", "DB connections"] }

# Update checklist
curl -X PATCH https://buildd.dev/api/workspaces/{id}/heartbeat \
  -H "Authorization: Bearer bld_xxx" \
  -H "Content-Type: application/json" \
  -d '{ "checklist": ["API health", "Error rate", "DB connections", "SSL certs"] }'
```
